## 📌 Overview

'초능력 회사 취업을 위한 전투 면접'이라는 콘셉트의 2D 액션 게임입니다.\
플레이어는 웨이브 형식으로 등장하는 적을 처치하고, 최종 보스(면접관)를
격파하여 높은 랭크를 달성하는 것을 목표로 합니다.

------------------------------------------------------------------------

## 🎯 Key Contributions

-   **Core Gameplay Systems 직접 설계 및 구현**
    -   공격, 웨이브, 사운드, 랭크 시스템 전체 개발
-   **Coroutine 없이 시간 제어 로직 구현**
-   **Queue 기반 구조 설계로 시스템 일관성 확보**
-   **애니메이션 이벤트 기반 히트박스 처리 구조 설계**
-   **디버깅 및 문제 해결 중심 개발 경험 축적**

------------------------------------------------------------------------

## ⚙️ Tech Stack

-   Engine: Unity 6.3
-   Language: C#
-   Architecture: Component 기반 설계
-   Data Structure: Queue, List 적극 활용
-   Tool: Aseprite
-   Art Pipeline: AI 생성 → 픽셀 아트 변환

------------------------------------------------------------------------

## 🧩 Core Systems

### 1. Combat System (공격 시스템)

-   입력 허용 시간 기반 콤보 공격 구현
-   Queue를 활용한 입력 버퍼 처리
-   애니메이션 이벤트를 통해 히트박스 생성
-   공격 중 이동 제한 + 점프를 통한 캔슬 설계

👉 핵심 포인트: - 입력 → Queue → 순차 실행 구조로 안정적인 콤보 처리

------------------------------------------------------------------------

### 2. Wave System (웨이브 시스템)

-   Inspector 기반 웨이브 데이터 관리
-   Queue를 활용한 적 스폰 순서 제어
-   일정 시간 간격 스폰 로직 구현
-   보스 / 엘리트 / 일반 몬스터 분기 처리

👉 핵심 포인트: - 데이터 기반 설계 + Queue 활용으로 확장성 확보

------------------------------------------------------------------------

### 3. SFX Timer System

-   List + Queue 기반 반복 재생 구조
-   Coroutine 없이 시간 기반 사운드 제어 구현

👉 핵심 포인트: - 타이머 및 반복 로직을 직접 구현하여 엔진 의존도 감소

------------------------------------------------------------------------

### 4. Rank System

-   적 처치 수 기반 점수 계산
-   S / A / B 등급 평가 시스템 구현
-   랭크 달성 시 연출 추가

------------------------------------------------------------------------

### 5. Random Stage System

-   Inspector에서 관리되는 스테이지 리스트 기반
-   랜덤 스테이지 진입 시스템 구현

------------------------------------------------------------------------

## 🐞Troubleshooter

### 1. Tilemap Collider 충돌 문제

-   문제: 타일마다 충돌 → 이동 끊김 발생
-   해결: Collider 구조 변경 (Capsule + Box)

### 2. 히트박스 빌드 오류

-   문제: Editor에서는 정상, 빌드에서 Trigger 미작동
-   해결: Initialize 시점에서 Collider 설정

### 3. 사망 상태에서 공격 발생

-   문제: 상태 전이 충돌 (Attack → Death)
-   해결: 공격 로직에 상태 조건 추가

### 4. 시간 정지 스킬 버그

-   문제: 특정 상황에서 시간 복구 실패
-   해결: 히트박스 의존 로직 제거 → 독립 시스템으로 분리
